
# AI Ethics considerations

## 8. Ethics in AI Engineering {#ethics-in-ai-engineering}

### 8.1 Core Ethical Principles

**Fairness and Non-discrimination:** LLMs trained on biased data can perpetuate and amplify societal biases. AI engineers must actively measure and mitigate bias in both training data and model outputs.

```python
# Bias detection prompt for testing
def test_for_demographic_bias(client, domain: str, test_cases: list[dict]) -> dict:
    """
    Test an AI system for demographic bias by presenting equivalent scenarios
    differing only by demographic characteristics.
    
    test_cases: list of {"name": str, "prompt": str, "demographic": str}
    """
    results = {}
    
    for case in test_cases:
        response = client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=200,
            messages=[{"role": "user", "content": case["prompt"]}]
        )
        results[case["demographic"]] = {
            "prompt": case["prompt"],
            "response": response.content[0].text,
            "response_length": len(response.content[0].text)
        }
    
    # Flag if response lengths differ significantly (a simple proxy for bias)
    lengths = [v["response_length"] for v in results.values()]
    if lengths and (max(lengths) - min(lengths)) / max(max(lengths), 1) > 0.5:
        print(f"WARNING: Significant response length disparity detected — potential bias in '{domain}'")
    
    return results
```

**Transparency and Explainability:** Users interacting with AI systems have a right to know they are doing so. Where AI is involved in significant decisions, the reasoning should be explainable.

```python
def add_transparency_disclosure(response: str, is_ai_generated: bool = True) -> str:
    """Append a standard AI disclosure to responses."""
    if is_ai_generated:
        disclaimer = "\n\n---\n*This response was generated by an AI system. It may contain errors and should not substitute for professional advice.*"
        return response + disclaimer
    return response
```

**Human Oversight (Human-in-the-Loop):** For high-stakes decisions, always build in human review checkpoints.

```python
from typing import Callable, Optional

class HITLController:
    """Human-in-the-loop controller for AI agent actions."""
    
    HIGH_RISK_ACTIONS = {
        "send_email", "delete_file", "execute_code", "database_write",
        "financial_transaction", "medical_recommendation", "legal_document_generation"
    }
    
    def __init__(self, review_callback: Optional[Callable] = None):
        self.review_callback = review_callback or self._default_review
        self.pending_actions = []
    
    def _default_review(self, action: str, context: dict) -> bool:
        """Default: require explicit approval for all high-risk actions."""
        print(f"\n[HUMAN REVIEW REQUIRED]")
        print(f"Action: {action}")
        print(f"Context: {context}")
        approval = input("Approve? (yes/no): ").strip().lower()
        return approval == "yes"
    
    def check_action(self, action_name: str, context: dict) -> bool:
        """
        Returns True if action is approved to proceed.
        """
        if action_name in self.HIGH_RISK_ACTIONS:
            approved = self.review_callback(action_name, context)
            self.pending_actions.append({
                "action": action_name,
                "context": context,
                "approved": approved,
                "timestamp": __import__("datetime").datetime.utcnow().isoformat()
            })
            return approved
        return True  # Low-risk actions auto-approved
```

### 8.2 Hallucination and Misinformation Risk

LLMs can confidently generate false information. In high-stakes domains (healthcare, legal, finance), this is an active safety concern.

**Mitigations:**
- Implement RAG with citations to ground responses in verified sources
- Add confidence disclaimers to outputs, especially in high-risk domains
- Use fact-checking agents as a secondary validation step
- Restrict LLM scope to only what it demonstrably knows (avoid open-ended generation for facts)

### 8.3 Environmental Considerations

Training and running LLMs has a measurable carbon footprint. As a responsible AI engineer:
- Prefer smaller, distilled models for tasks where a large model is not required
- Cache frequent inference results to reduce redundant compute
- Choose cloud providers with renewable energy commitments for AI workloads
- Report on energy usage as part of AI governance practices

### 8.4 Responsible Disclosure

If you discover security vulnerabilities in LLM systems:
- Follow responsible disclosure practices (report to the vendor first)
- Many major AI providers including OpenAI, Anthropic, and Google have **bug bounty programs** for AI security research
- Do not publicly release working exploits without giving vendors time to patch
